## 02 — AI delivery model

<p align="center">
  <picture>
    <source media="(prefers-color-scheme: dark)" srcset="./assets/delivery-dark.svg" />
    <img src="./assets/delivery-light.svg" width="100%" alt="AI-directed engineering delivery model" />
  </picture>
</p>

| Principle | Application |
|---|---|
| **Read before changing** | Inspect source, current behavior, history, architecture boundaries, and existing constraints. |
| **Map before building** | Separate hard requirements, risks, dependencies, affected surfaces, and verification steps. |
| **Direct rather than delegate blindly** | Use AI for navigation, comparison, implementation, debugging, documentation, and test design while retaining technical judgment. |
| **Verify meaningful claims** | Review diffs and run type checks, tests, builds, logs, simulators, and physical-device validation where required. |
| **Preserve stable systems** | Improve targeted areas without rewriting known-good architecture for novelty. |
| **Ship with evidence** | Harden behavior, document decisions, disclose limitations, and leave the codebase easier to continue. |

This is how I coordinate AI across large codebases while keeping output **traceable, constraint-aware, technically grounded, and production-focused**.
